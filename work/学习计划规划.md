## 六、Android 框架学习方法详解

### 方法 1：结合 hiview apk 代码实践学习

**步骤 1：下载 hiview 源码后分析组件注册**
```bash
# 查看 Manifest 了解组件注册
cat app/src/main/AndroidManifest.xml

# 查找 AMS 相关调用
grep -r "ActivityManager" app/src/

# 查找 PMS 相关调用  
grep -r "PackageManager" app/src/

# 查找 WMS 相关调用
grep -r "WindowManager" app/src/
```

**步骤 2：理解 hiview 与系统服务的交互方式**
- hiview 如何与 AMS 通信获取其他应用信息？
- hiview 如何查询已安装应用列表（PMS）？
- hiview 的窗口如何被 WMS 管理？

### 方法 2：使用 adb dumpsys 命令观察服务状态

```bash
# AMS 相关
adb shell dumpsys activity activities    # 查看所有 Activity 栈
adb shell dumpsys activity history       # 查看历史 Activity
adb shell dumpsys activity processes     # 查看进程信息

# PMS 相关
adb shell pm list packages               # 查看已安装包列表
adb shell dumpsys package com.example    # 查看特定包详细信息
adb shell pm list permissions -g         # 查看权限授予情况

# WMS 相关
adb shell dumpsys window windows         # 查看所有窗口信息
adb shell dumpsys window | grep "mCurrentFocus"  # 查看当前焦点窗口
adb shell dumpsys window animator        # 查看窗口动画状态

# IMS 相关
adb shell getevent -l                    # 查看输入设备列表
adb shell dumpsys input                  # 查看 IMS 状态

# 模拟输入事件（测试用）
adb shell input tap 100 200              # 点击
adb shell input swipe 100 100 200 200    # 滑动
adb shell input keyevent 3               # HOME 键
```

### 方法 3：AOSP 源码阅读路径

**核心服务源码位置**：
```
frameworks/base/services/core/java/com/android/server/am/
  ├── ActivityManagerService.java    # AMS 主类
  ├── ActivityStack.java             # Activity 栈管理
  ├── ProcessRecord.java             # 进程记录
  └── ActivityRecord.java            # Activity 记录

frameworks/base/services/core/java/com/android/server/pm/
  ├── PackageManagerService.java     # PMS 主类
  ├── PackageParser.java             # APK 解析器
  └── Settings.java                  # 包管理设置

frameworks/base/services/core/java/com/android/server/wm/
  ├── WindowManagerService.java      # WMS 主类
  ├── WindowState.java               # 窗口状态
  └── AppWindowToken.java            # 应用窗口令牌
```
