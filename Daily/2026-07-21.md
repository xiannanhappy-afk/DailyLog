### 🔧 自己 pull report.pb 的可复现步骤

**前提：** 测试机已 root、adb 在 PATH（本机 adb 路径：/d/30_dev/Android/SDK/platform-tools/adb）、用 Git Bash 跑命令。

**场景一：config 已推过、statsd 在攒数据（只 dump）**

```bash
# 1. dump（configId 固定 29573019807062071，必须 | cat 绕过 tty 检测）
adb shell "cmd stats dump-report 0 29573019807062071 --keep_data --proto 2>&1 | cat > /data/local/tmp/r.pb"
# 2. 看大小（<1000B 基本空，>10KB 才有数据）
adb shell "ls -la /data/local/tmp/r.pb"
# 3. pull（Git Bash 必须 MSYS_NO_PATHCONV=1）
MSYS_NO_PATHCONV=1 adb pull /data/local/tmp/r.pb ./report.pb
```

**场景二：config 没了/要重新采（完整流程）**

```bash
# 1. 清旧 config
adb shell "cmd stats config remove 0 29573019807062071"
# 2. 推 config（必须 pipe 不能 < 重定向，否则 SELinux denied read）
adb shell "cat /data/system/hiview/statsd/config_memory_distribution_beta.pb | cmd stats config update 0 29573019807062071"
# 3. 保持目标 app 前台运行，等 60 秒（bucket_size=60s，第一个 bucket 关闭才有数据）
# 4. dump + pull（同场景一）
adb shell "cmd stats dump-report 0 29573019807062071 --keep_data --proto 2>&1 | cat > /data/local/tmp/r.pb"
MSYS_NO_PATHCONV=1 adb pull /data/local/tmp/r.pb ./report.pb
```

**三个必须记住的坑：**

| 坑 | 现象 | 对策 |
|---|---|---|
| SELinux | config update 用 `< file` 报 avc denied read / Failed transaction | 改用 `cat file \| cmd stats config update` |
| tty 检测 | dump-report 报 Failed transaction | 输出加 `\| cat` |
| bucket 没满 | dump 出 21B 或 BUCKET_TOO_SMALL、无 app 数据 | 推完 config 等 ≥60 秒再 dump；要采 app 就让它一直跑 |

**命令里固定值说明：**
- `0` = uid（root 推的 config 用 0；hiview 自推的是 1000 但采集链坏了别用）
- `29573019807062071` = configId，从 /data/system/hiview/statsd/statsd_config.json 读，固定不变
- `--keep_data` = dump 后不清空（可反复 dump）
- `--proto` = 输出 protobuf 格式

- 解析工具：Z:\z00037305\test\Statsd\python_parse\version_3\pb_to_txt.py（用法：python3 pb_to_txt.py report.pb statsd_tool out.txt --debug）

### 📝 adb shell 两种写法对比（为什么用引号包整条）

**写法 A（引号包整条，管道在设备端执行）——推荐，可脚本化：**
```bash
adb shell "cmd stats dump-report 0 29573019807062071 --keep_data --proto 2>&1 | cat > /data/local/tmp/r.pb"
```

**写法 B（adb shell 后逐条输入，交互式）——适合临时调试：**
```bash
adb shell
$ cmd stats dump-report 0 29573019807062071 --keep_data --proto 2>&1 | cat > /data/local/tmp/r.pb
$ exit
```

**两种等价，都在设备端执行 `| cat >`，都能绕过 tty 检测。**

**为什么不用写法 C（引号外有管道，错误）：**
```bash
adb shell cmd stats dump-report 0 ... | cat > ./r.pb    # ← 不加引号，管道跑在本机
```
`|` 和 `>` 跑在电脑本机，`| cat` 失去绕过设备端 tty 检测的作用，会触发 `Failed transaction`。

**要点：**
- `| cat` 必须在设备端执行才有效（tty 检测在手机里的 cmd stats 触发）→ 写法 A/B 都满足
- `2>&1` 和 `> /data/local/tmp/r.pb` 是设备路径重定向，也必须在设备端 → 引号包整条才保证
- 写法 A：单行可复制粘贴、可写脚本，最省事
- 写法 B：边敲边看、临时探索，但需手动 exit
- 写法 C：千万别用，管道跑本机导致 `| cat` 失效
