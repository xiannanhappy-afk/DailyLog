**问题解决：**
- `elink-cli` 在 Git Bash 中无法找到 → 根因：Git Bash 默认 `MSYS2_PATH_TYPE=minimal` 不继承完整 Windows PATH，且 `~/.bashrc` 中硬编码的 PATH 缺少 `/c/Users/z00037305/elink-cli`；修复：在 `.bashrc` PATH 最前面添加 elink-cli 目录
- `MSYS2_PATH_TYPE=inherit` 设置后仍无效 → Claude Code 启动 bash 时传入的初始 PATH 就不含完整系统 PATH，需在 `.bashrc` 中手动追加
- elink-cli 代理警告 → 设置 `LARK_CLI_NO_PROXY=1` 系统变量（注意变量名是 `LARK` 不是 `ELINK`，旧版遗留命名）

**配置调整：**
- `~/.bashrc`：PATH 添加 `/c/Users/z00037305/elink-cli`
- `MSYS2_PATH_TYPE=inherit`（系统变量）
- `LARK_CLI_NO_PROXY=1`（系统变量，仅禁用 elink-cli 代理）
